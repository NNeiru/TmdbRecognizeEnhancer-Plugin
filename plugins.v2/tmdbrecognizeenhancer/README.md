# 媒体整理增强

“媒体整理增强”在 MoviePilot 已有识别和整理链路之上提供可选扩展。插件的原则是：优先复用 MP 结果、只处理已启用模块、无法唯一判断时安全跳过。

## 运行要求

- MoviePilot V2 `>= 2.13.0`
- 插件版本：`0.8.5`
- 集数偏移在最终媒体识别成功后执行，兼容 MoviePilot 的“插件优先”和“原生优先”
- Emby 剧集组联动需要 MP 已配置 Emby，并在 Emby 安装支持 `TmdbEg` 的 StrmAssistant

插件不修改 MP/Rust 文件。运行时适配器若检测到上游接口不兼容，会在“状态与开关”中标明不可用能力。

## 处理顺序

一次实际整理中，各能力按以下逻辑衔接：

1. MoviePilot 识别词和 Rust 元信息解析先处理原标题。
2. 插件读取解析后的标题、年份、媒体类型、制作组和季集坐标。
3. 字段覆盖和制作组类型只补充候选判断所需的上下文。
4. 按 MoviePilot 设置执行插件优先或原生优先识别；插件候选无法安全接纳时返回原整理链。
5. 最终 TMDBID 确定后，无论结果来自原生还是插件，命中维护规则都会执行集数偏移。
6. 制作组编排、自定义 Jinja2 字段和 MP 命名模板生成首次命名。
7. 文本映射对完整相对路径执行最终二次渲染。
8. 整理完成后，只有本次实际采用剧集组的任务才进入可选的 Emby 联动队列。

“综合试跑”按同样的顺序展示判断依据和最终命名，但不请求文件整理，也不写入 Emby。

## 模块说明

### 状态与开关

集中管理插件总开关、TMDB 搜索增强、集数偏移、字段与制作组、命名规则、Emby 入库联动和侧栏入口。模块关闭后不会参与实际整理；保存后页面会显示后端实际生效状态。

### TMDB 搜索增强

支持两种独立模式：

- **TMDB 单次首结果**：完整解析标题只查询一次，在符合明确类型约束的结果中采用首项，不使用最低分或领先分差。
- **可解释评分**：综合标题/token 相似度、MP 原标题锚点、年份、媒体类型、季存在性、查询来源和 TMDB 排名；第一名同时达到最低分和领先分差才接纳。

候选召回会保留 TMDB Multi Search 的原始排序；已明确电影或电视剧类型时，再合并 Movie/TV 专用搜索补回 Multi 首页可能遗漏的同类候选。补充策略包括主体名称降级、逐词缩短、候选别名、近期季度目录证据、近期重复命中和可选的搜索引擎交叉验证。季度窗口可覆盖最近 1–4 季，默认读取当前与上一季度的缓存目录；降级候选仍需与未缩减标题产生关联，避免越缩越宽导致误识别。

TMDB 排除名单会在评分、排序和分差前剔除指定候选；优先名单仅在本次搜索实际返回指定 ID 时直接选择。两者冲突时以排除为准。

明确的电视剧类型或季集坐标会约束候选类型，防止同名电影、OVA 或真人条目截胡。S02 及后续季度文件携带的年份只视为本季播出上下文，不与 TMDB Series 的首播年份进行强匹配。

“使用 MP 共享识别”默认开启，但只在 TMDB Multi 与类型专用搜索均无候选时查询，并继续接受类型、制作组题材和 TMDB 排除名单检查。共享数据由 MoviePilot 中央服务维护，插件只提供开关和只读命中诊断，不提供虚假的本地编辑页；人工纠错请使用 TMDB 优先/排除名单。搜索引擎功能默认关闭；启用后只把具体 TMDB 直链和候选共现作为辅助证据，不直接决定结果。

### 集数偏移

该模块包含三个子页面：

#### 偏移规则维护

- 每个 TMDBID 维护一个目标编集：TMDB 默认编集或指定 `episode_group_id`。
- 输入可以来自默认季集、Production/Absolute 剧集组坐标或全局累计集数。
- 插件通过 TMDB Episode ID 和目标分季结构转换坐标；已经符合目标时保持不变。
- Special/OVA/番外优先映射到 S00，不计入正片累计顺序。
- 目标坐标不唯一、跨度跨季或缺少关键 TMDB 数据时安全跳过。
- TMDB 暂缺最新集时，只在已命中同一季度片段、目标末季且缺口很小的条件下保守推算。

原有 MoviePilot 自定义识别词仍优先生效。若词表已经得到正确季集，插件不会重复偏移。

#### 季度番剧看板

- 使用 AniList 建立季度目录，并通过 AniBridge、Bangumi 和 TMDB 补充标题、地区、载体和跨库映射。
- 看板 TMDB 匹配使用独立的多别名策略，不继承实际整理中的单次搜索额度。
- 支持平铺、列表和紧凑视图，以及季度、地区、载体、续作、扫描状态和标题筛选。
- 单条或批量建立规则；失败条目可补充 TMDBID、重试或忽略。
- “优先剧集组”只在剧集组更符合主流分季时采用，避免把原本正确的多季默认编集换成累计或平台特有顺序。

#### Emby 剧集组联动

- 仅处理本次整理实际采用剧集组的电视剧；仅维护规则不会触发写入。
- 复用 MP 的 Emby 地址、用户和 API Key，不保存额外凭据。
- 先按 TMDBID 查询 Series；同 ID 多条目时再用 MP 最终文件路径和 Emby Series 路径定位。
- 容器路径不同可设置最长前缀映射；仍有歧义时停止并列出候选。
- 可显式对全部同 TMDBID Series 批量写入，每条仍独立执行冲突保护并返回结果。
- `TmdbEg` 已相同则幂等跳过；值不同默认报告冲突，用户可选择由维护规则覆盖。
- 整理早于 Emby 入库时，任务会持久保存并按设置重试；MP 重启后可继续。

### 字段与制作组

- 浏览 MP 当前进程加载的 `releaseGroup`、`resourceType`、`effect`、`videoFormat`、`videoCodec`、`videoBit`、`audioCodec`、`fps`、`webSource`、`customization` 和季集等规则。
- 对内置字段的修改保存在插件覆盖层，不写入 MP/Rust 文件，可恢复原值。
- 制作组可简单标记为动漫、真人电视剧或不参与判断；明确分类会在 TMDB 选择前排除题材冲突候选，缺少题材信息的候选会保守保留。
- 制作组还可按组补充 `resourceType`、`webSource`、编码、位深、特效和 `customization` 等命名字段；默认只补 MP 空值，合作组给出冲突值时自动跳过。
- 自定义 Jinja2 字段可读取 MP 命名阶段真实存在的媒体、文件、源路径和分类目标目录上下文，并支持字段依赖、回退值和试算。

制作组分类属于用户明确设置的题材硬约束。综合发布组或证据不足的组应保持“不参与判断”，避免根据个人下载偏好形成错误类型结论。

### 媒体信息识别

- 整理前通过容器内 `ffprobe` 读取实际分辨率、编码、位深、帧率、HDR/杜比视界及音轨、内封字幕语言；扫描结果在 MP 命名模板渲染前注入并按文件修改时间缓存。
- ffprobe 始终读取完整的 streams、format 和 chapters；每类开关只决定哪些结果进入 MP/Jinja 命名上下文，并设置补空、覆盖或追加策略，不会裁剪神医联动使用的原始扫描数据。字幕组合写入 `customization`（`probe_subtitle_mapped`）：规则按行序首条命中，`简体+繁体 => 简繁内封` 精确匹配语言集合，`包含:简体+日语 => 简日内封` 为子集匹配，`>=4 => 多国字幕` 按语言数量（适合 CR/Netflix 多语种 WEB-DL）；未命中任何规则时自动回退为语言组合（如“简繁日内封”，超过 3 种语言归为“多国内封”），因此只需为想改名的组合写规则。
- 字幕相关变量分工：`probe_subtitle_languages` 是轨道 language 标签对应的单一语言（标题只用于细分简体/繁体，“简日双语”轨记为简体）；`probe_subtitle_titles` 是轨道原始标题（简日双语、繁日雙語）；`probe_subtitle_track_labels` 是每轨归一后的简写（简日、繁日）。简繁判定覆盖“简日双语”“繁日雙語”“GB/BIG5/zh-Hans/zh-Hant”等常见标注写法，映射规则的“简体+繁体+日语”与“简日+繁日”写法可互相命中。
- 文件试扫可通过 MoviePilot 容器目录树选择真实媒体文件，同时查看写入 MP 标准字段的结果和全部 `probe_*` Jinja2 变量。
- 可选的「神医媒体信息联动」仅适配 StrmAssistant **Pro**：整理前 ffprobe 的完整原始结果在 `TransferComplete` 后转换为 Emby `MediaSourceInfo`，通过 Pro 注册的 `POST /Items/SyncMediaInfo` 接口推送；复用 MP 已配置的 Emby 凭据，支持按服务器路径映射、后台等待/重试、任务持久化和单文件真实试推。路径映射左侧填写 MP 整理后真实媒体文件的目录前缀，右侧填写 Emby 条目自身媒体文件的目录前缀；STRM 库填写 `.strm` 文件所在目录（不是 `.strm` 内容中的真实媒体地址），插件会自动把目标扩展名换成 `.strm`。它不生成 `-mediainfo.json`、不依赖神医的媒体信息持久化开关，也不调用 ffmpeg；社区版未提供该接口，因此不受支持。
- ISO 原盘探测（可选）：官方镜像的 ffprobe 没有 libbluray，读不出 ISO 原盘播放列表。开启后插件从 [StrmAssistant.Releases](https://github.com/sjtuross/StrmAssistant.Releases/tree/main/static-ffprobe) 自动下载对应平台的静态 ffprobe（约 15MB）到插件数据目录并以 `bluray:` 协议探测，**只接管 `.iso` 文件**，普通视频仍使用原 ffprobe。下载遵循 MP 的 `GITHUB_PROXY` 与代理设置；默认关闭，不开启则完全不下载。
- `probe_*` 可直接用于 MP 命名模板，但只有启用本模块且实际文件可读时才有值；无法从文件名推断的音轨或字幕信息保持为空。

当前 MoviePilot 官方 Dockerfile 已将静态 `ffmpeg` 与 `ffprobe` 放在 `/usr/local/bin`。若扫描页显示不可用，应先确认运行的是当前官方镜像并重新创建容器；自定义镜像可持久挂载二进制并在插件中填写其容器内路径。插件不会在运行中自动执行包管理器或下载程序，因为插件通常没有 root 权限，容器重建也会丢失这类临时安装。

字幕扫描只识别容器中存在独立字幕轨的内封字幕。已经烧录进视频画面的硬字幕没有可供 ffprobe 读取的字幕流，需要 OCR 才可能判断，因此本模块不处理。

### 命名规则

命名规则按以下顺序执行：

1. 单组制作组规则（别名、最终名称、固定位置、排序和连接符）。
2. 标题中已有的制作组连接符或全局默认连接符。
3. MP Jinja2 命名模板与自定义字段渲染。
4. 对完整相对路径执行按优先级排列的字面或正则文本映射。

单组专属连接符的优先级最高。全局开关只决定“标题原连接符”和“默认连接符”谁优先，不会覆盖单组设置。文本映射可统一处理标题、目录和字幕结果，例如：

```text
命运／奇异赝品 => 命运-奇异赝品
\.chi\.zh-cn => .chs
\.zh-tw => .cht
```

路径结果包含绝对路径、父目录跳转或其它越界形式时会被拒绝。

### 综合试跑、日志与性能诊断

- **综合试跑**：展示 MP 解析、字段覆盖、制作组证据、TMDB 候选、集数偏移、制作组编排和最终命名。
- **日志**：统一记录各模块的决策摘要，不保存 TMDB 完整响应或 Emby 凭据。
- **性能诊断**：用户主动开始后短周期采样 MP 进程、整机负载、插件缓存、API 往返、浏览器 FPS、DOM 和 JS 堆；停止或离开页面后不继续采样。

## 网络与数据

TMDB 搜索、季度目录和可选搜索引擎功能需要访问相应外部服务，并复用 MoviePilot 系统代理设置。插件只持久保存用户配置、偏移规则、有限的候选摘要、近期识别统计和 Emby 联动任务，不保存完整 TMDB 响应或媒体服务器密钥。

## 常见问题

### 已启用插件但集数没有偏移

确认插件总开关与集数偏移模块均已启用、运行时适配显示兼容，并检查最终 TMDBID、来源季集和维护规则是否齐全。是否开启 MoviePilot“识别插件优先”只决定谁先选择 TMDB，不再决定集数偏移是否执行。综合试跑会明确显示“未维护规则”“缺少坐标”或“无法唯一映射”。

### 同一 TMDBID 在 Emby 中存在多个 Series

优先配置 MP 路径前缀到 Emby 路径前缀的映射，并用实际整理后的媒体文件路径试跑。若跨库副本都应使用同一剧集组，可使用显式“全部修改”；否则插件不会自行猜测。

### 更新后仍显示旧页面

每个版本使用独立的 `dist/ui/v版本号` 入口，并会在前后端版本不一致时自动重载一次。若浏览器仍保留升级前已打开的页面，关闭插件窗口后刷新 MoviePilot 页面即可，不需要清除插件数据。

### 配置保存后似乎没有生效

以“状态与开关”显示的后端状态为准。Emby 联动、路径映射和任务参数从 `0.6.0` 起与主配置合并保存，不会再被旧页面快照覆盖。

## 升级与回退

MoviePilot 从 GitHub Release 安装对应版本 ZIP。正式版本可从插件版本历史中选择安装；回退不会自动删除新版本产生的规则或队列。跨大版本回退前建议先备份 MoviePilot 配置目录。
