# TMDB 识别增强

在 MoviePilot 原生媒体识别失败后，生成有限的降级搜索词，从 TMDB 搜索结果中选择可信候选，再把标准名称交回原生识别和整理链。

## 默认策略

- 保留原生完全匹配的最高优先级
- 辅助识别收到原标题后，优先复用 MoviePilot 识别词与 Rust 解析出的名称、年份、类型和季集信息
- 完整标题失败后，可尝试冒号等分隔符前的主体名称
- 对候选的查询来源单独评分：完整标题唯一命中最高，宽泛降级词引入的候选可信度更低，多次重复命中会获得小幅加权
- 可选搜索引擎兜底：仅在 TMDB 直接候选未过阈值时，通过 DDGS 搜索并发现 TMDB 链接
- 搜索结果必须同时支持罗马音标题和候选英文/日文别名的共现关系，单独一条 TMDB 链接不会被直接接纳
- 拉取候选别名和译名，按 TMDB ID 去重
- 综合 token 覆盖、字符相似、主体前缀、搜索排名、年份、类型和季是否存在进行评分
- 第一名必须同时达到最低得分和领先分差，模糊结果会被安全拒绝

插件使用 Vue 联邦页面，提供策略设置、单条标题试跑、候选得分解释和最近运行记录。当前版本不修改 MoviePilot-Rust。

试跑框既可接收 MoviePilot 已提取标题，也可直接粘贴完整原始文件名；插件会展示解析后的标题和实际提交给 TMDB 的降级搜索词。

搜索引擎兜底默认关闭。启用后，解析标题会发送给所选搜索引擎，并自动使用 MoviePilot 的系统代理；返回内容只保存结果数量、发现的 TMDB ID 和最终证据摘要。

“TMDB 首轮优先”预设只执行一次完整标题的 `/search/multi`：唯一结果的查询来源分为 100；存在多个结果时第一名为 90、第二名为 82。该预设把其他可调评分权重和季缺失扣分设为 0，并关闭搜索词降级与搜索引擎兜底。
