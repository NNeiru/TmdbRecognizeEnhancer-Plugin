# TMDB 识别增强

从 TMDB 搜索结果中选择可信候选，并可在确定 TMDBID 后把动漫片源季集归一化到 TMDB 默认编集或用户选择的剧集组。

## 默认策略

- 默认使用独立的“TMDB 单次首结果”模式：只提交一次完整解析标题并采用第一个影视结果，不使用评分或领先分差。
- 可切换到“可解释评分”模式，由标题、MP 原标题、年份、类型、排名、查询来源和阈值共同决策。
- 保留原生完全匹配的最高优先级
- 辅助识别收到原标题后，优先复用 MoviePilot 识别词与 Rust 解析出的名称、年份、类型和季集信息
- 完整标题失败后，可尝试冒号等分隔符前的主体名称
- 对候选的查询来源单独评分：完整标题唯一命中最高，宽泛降级词引入的候选可信度更低，多次重复命中会获得小幅加权
- 可选搜索引擎交叉验证：自动模式固定单一后端并缓存同一查询，只接受具体 TMDB 直链与原标题/候选别名的共现证据
- 拉取候选别名和译名，按 TMDB ID 去重
- 综合 token 覆盖、字符相似、主体前缀、搜索排名、年份、类型和季是否存在进行评分
- 评分模式下，主体降级和逐词缩短产生的候选还必须与未缩减标题或 MP 原标题保持锚点关联；第一名同时达到最低分和领先分差后才会接纳

插件使用 Vue 联邦页面，提供策略设置、单条标题试跑、候选得分解释和最近运行记录。当前版本不修改 MoviePilot-Rust。

## 集数归一化

- MoviePilot 自定义识别词仍先执行；插件通过运行时上下文读取处理后的标题、季和集。
- 每个 TMDBID 只维护最终目标：`TMDB 默认` 或一个 `episode_group_id`。
- 当前坐标已经符合目标时保持不变，避免对传统词表已经偏移好的结果重复处理。
- 输入可通过 TMDB 默认坐标、Production/Absolute 剧集组坐标或全局集数定位到同一 Episode ID。
- “第三期 - 01”这类歧义标题可用季度条目别名和目标起始季集确定实际片段。
- 无法得到唯一目标、批量范围跨越目标季或目标集不存在时安全跳过。
- 季度看板会随年份/季度选择自动加载，只展示当前季度，并可按地区、载体、续作特征和标题筛选。
- 单条加入与批量加入都会自动完成 TMDB 匹配；批量可选优先默认编集或优先 Production/Season 剧集组。
- 自动匹配失败的条目会集中列出，可补充 TMDBID 后重试，也可直接放弃。
- 归一化试跑只需粘贴原标题，季集和 TMDBID 均由插件自动解析与匹配。
- 季度目录使用 AniList，在加载阶段即可按国家、载体和续作关系筛选；AniBridge 映射优先提供 TMDBID，缺失时再并发执行结构化标题快速匹配。
- 失败响应不会再被标记为已维护；维护规则固定显示在季度看板之前。
- TMDB 补充扫描在后台逐条完成，可按正在扫描、已匹配和失败状态筛选。
- 搜索前会去除季号、Part/Cour 等季度噪声；维护规则允许校正 TMDBID。
- 看板与规则标题优先使用 Bangumi/TMDB 中文名，单条与批量失败分别处理。
- 维护规则支持按季度分类、标题/别名/TMDBID 搜索及批量删除当前筛选结果。
- 可在看板外直接输入 TMDBID 建立规则，优先根据 TMDB 季首播日期自动推断季度。
- 季度目录固定使用独立的多别名 TMDB 查询额度，不继承整理识别的单次搜索或降级参数。
- “优先剧集组”会先保留已经正常分季的 TMDB 默认编集；仅对默认单季累计条目评估 Production/Season、覆盖率、分季共识和 Special 完整度。
- TMDB 或剧集组暂缺最新集时，只在已命中季度片段、同一目标末季且最多缺 4 集的条件下安全推算。
- Bangumi 与 TMDB Animation Discover 补充国漫、欧美动画；TMDB 候选结合 Animation 类型、媒体类型、原始语言、首播年份和多季结构评分。
- MOVIE 使用电影候选，ONA/OVA/SPECIAL 同时评估电视剧与电影；电影匹配只供看板展示，不建立集数归一化规则。
- 季度、地区、载体、扫描状态、搜索条件和维护规则筛选会保存在当前浏览器中。
- 剧集组中的 Special/OVA/番外/特典会强制归入 S00，并从正片累计顺序中排除；常规组和组内集数统一从 S01/E01 重新编号。
- 默认编集把正篇尾部特别篇继续编号时，会优先通过相同 Episode ID 映射到剧集组 S00；例如默认 E16 可映射为目标 S00E04，而之前的 E11 不会被重复偏移。
- 剧集组完整度仅比较正篇 Episode ID；缺少 S00 小剧场不会被误判为正片缺失，完整收录 Special 的同类剧集组仍会获得额外优先级。
- 编辑规则时可查看每个剧集组的完整分季卡片，包括名称、集数范围、首末播出日期、Special 和正片覆盖率。
- 季度看板和维护规则都支持平铺、列表与紧凑三种显示方式，并保存当前浏览器的选择。
- 综合试跑按顺序展示 MP 解析、TMDB 搜索增强和集数偏移的结论；日志统一记录两个运行模块。

集数归一化需要 MoviePilot 开启“识别插件优先”，否则原生识别成功时不会触发插件。插件不会改写 MoviePilot 容器源码；运行时适配器不兼容时会在页面给出警告并停用归一化。

试跑框既可接收 MoviePilot 已提取标题，也可直接粘贴完整原始文件名；插件会展示解析后的标题和实际提交给 TMDB 的降级搜索词。

搜索引擎兜底默认关闭。启用后，解析标题会发送给所选搜索引擎，并自动使用 MoviePilot 的系统代理；返回内容只保存结果数量、发现的 TMDB ID 和最终证据摘要。

“TMDB 单次首结果”是与评分体系完全分离的选择模式：别名、年份和评分仍可用于试跑诊断，但不会改变 TMDB 返回顺序。需要更严格的错误防护时可切换到“可解释评分”。
